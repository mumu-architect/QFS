以下是一套 Go 语言文件分片上传 + 断点续传 的完整实现，基于 HTTP 协议，支持分片拆分、并发上传、断点续传、完整性校验等核心特性，代码结构清晰，可直接复用或扩展。
核心设计
分片策略：按固定大小（默认 5MB）拆分文件，每片携带「文件唯一标识（MD5）+ 分片索引 + 总分片数」元信息；
断点续传：服务端记录已接收的分片，客户端上传前先查询已上传分片，仅传未完成部分；
并发上传：客户端多协程并行上传分片，提升效率；
幂等性：通过「文件 MD5 + 分片索引」唯一标识分片，避免重复上传；
完整性校验：文件合并后校验整体 MD5，确保上传无损坏。

断点续传测试
客户端上传过程中手动终止（如 Ctrl+C）；
重新启动客户端，会自动查询已上传分片，仅上传未完成部分；
最终所有分片上传完成后，触发合并，得到完整文件。
生产环境优化建议
##### 存储优化：
将临时分片和最终文件存储替换为云存储（如 OSS、S3），避免本地磁盘瓶颈；
定期清理过期临时分片（如超过 24 小时未合并的分片）。
##### 分布式支持：
上传状态存储从内存 map 替换为 Redis（用 SADD 记录分片索引，SMEMBERS 查询）；
服务端可部署多个节点，通过 Redis 共享状态，支持负载均衡。
##### 安全性优化：
增加身份认证（如 JWT、API Key），防止非法上传；
用 HTTPS 加密传输，避免数据泄露；
限制单个文件最大大小和分片数，防止恶意攻击。
##### 性能优化：
客户端支持分片预读和缓存，提升上传效率；
服务端使用 io.CopyBuffer 减少内存分配；
开启 HTTP/2，支持多路复用，提升并发性能。
##### 错误处理优化：
客户端增加分片上传重试机制（如失败后重试 3 次）；
服务端返回更详细的错误信息，方便排查问题。
该实现覆盖了文件分片上传 + 断点续传的核心场景，代码简洁易懂，可直接用于中小规模业务，根据生产需求调整配置即可。
